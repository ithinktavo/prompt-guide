Capital One Context (August 2025 - December 2025):
The client seemed to be more happy with the team and the work being done. I implemented new processes and improvements to improve the team's performance and satisfaction.
Created a new Agile framework internally and better story system for the team to follow. This improved the team velocity and issue where stories were not being completed in a sprint so there could be carry overs.

I made sure that each developer was responsible for their epic and that included being able to create the stories needed to complete the epic. 
This allowed the team to be more autonomous and not to rely so much on the client to create the stories (which was a big issue and the client made it clear in internal feedback).

My team name is Fraud Watchdogs and I was the Tech Lead for this team. Its composed of me the tech lead, Ana (senior developer), Moya (junior developer), Juquan (senior developer) and Arpita (junior developer) and recently in place of Arpita, we have Jerry (junior developer).

In 2025 the main deliverables were:
- Third Party ACH Migration Moving from Rules (9 rules) as an Engine (Watchtower OL) to Watchtower Platform.
    - This was a big project that involved a lot of work and was the main priority for the team.
    - As part of this process, python rules needed to be migrated or converted to DMN rules.
    - One of the challenges here are the data elements and gathering and transforming the data that we need to use for the DMN rules.
    - Since this is a service that makes a decision on ACH transactions, this service in the new platform needs to be analyzed closely and gather data comparing the legacy system with the determination of the new platform.
- Migration of Java Springboot service on ECS to a Lambda function.
    - This was a challenge because the client thought that it was going to be a simple migration but it was not. This was to the fact that they thought this would be a copy and paste of the Java Springboot code to the lambda.
    - You need to tweak the code to make it work within the context of a Lambda function. Like for example the reactive code (returning responses before actually finishing processing), ephemeral storage, etc.
- HashiCorp Vault Secrets Migration to AWS Secrets Manager.
    - Goal is to remove dependency on HashiCorp Vault and move to AWS Secrets Manager.
    - We had to create the current secrets in AWS Secrets Manager and then update the different code applications to use the new secrets manager. And we had to do this for Dev, QA and Prod environments.

Additionally as part of being the Tech lead I am responsible for providing weekly status reports to the client and the Accenture team. I also need to provide Quarterly status reports to the client and the Accenture team. And to continue mentoring and guiding others and my team members.

By the end of 2025, the team was in green status and the client was satisfied with the work and progress made.

The lambda migration was finished, were were able to delete the ECS service, and lambda was serving production traffic.  The last remaining item is finalizing Splunk alerting. (I also worked on this effort and Juquan (another senior developer from my team))

The Secrets migration is almost finished, we updated the code applications within our scope to read secrets from AWS Secrets Manager and we see from Vault that the for production are not being used based on last accessed date. We are investigating which application is using secrets from HashiCorp Vault in dev. Once that is identified, we will delete vault dev and qa and finish with production, but the heavy lifting is done.  (I worked on this effort and Juquan)

Third Party ACH Migration most of the heavy lifting is done. We are analyzing mismatches between legacy system and new platform so that we can get signoff from Fraus Strategy team to start serving real traffic and being authoritative which means that the decision made in the new platfrom is the one that is going to be followed. After signoff, we will allow only 2% of the traffic to go to the new platform and we will monitor the results and validate post decision processing. If those are good, we will in a course of a day increment to 100%. After that, then there needs to be a decommissioning of the legacy system, which not sure if that would be our responsibility or the client's. (Ana and Moya 2 developers worked on this effort)
